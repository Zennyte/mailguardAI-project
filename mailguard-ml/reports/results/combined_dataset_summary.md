## Class Imbalance Note

The dataset has a moderate class imbalance:

- **safe** is the largest class at 57.8%, driven mainly by CEAS_08 and Enron.
- **spam** is the second class at 35.0%.
- **phishing** is the smallest class at only 7.1%, contributed entirely by the
  Nazario and Nigerian_Fraud datasets.

This imbalance is expected and acceptable for this project. To prevent the
classifiers from ignoring the phishing class during training, all classifiers
that support it must use `class_weight='balanced'`. This instructs the model
to give higher weight to underrepresented classes during training.

Phishing detection is the most critical use case in this project, so it is
important that the model does not under-predict it.

## Note on the Processed CSV

The file `data/processed/email_dataset_combined.csv` is **not committed to
GitHub**. It is listed in `.gitignore` because:

- Dataset files are large and not suitable for version control.
- The raw source datasets are also not committed for the same reason.
- The processed CSV can always be regenerated by running:

```
python src/prepare_dataset.py
```

Anyone setting up the project locally must first place the raw CSV files in
`data/raw/` and then run the preparation script.
